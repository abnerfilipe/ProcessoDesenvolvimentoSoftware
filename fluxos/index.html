A seguir, algumas abordagens para designação dessa função: Designe as funções de Analista do Processo de Negócios e de Arquiteto de Negócio (e, às vezes, Designer de Negócio) para a mesma pessoa. Essas funções interagem constantemente, portanto, é possível que seja mais eficiente que a mesma pessoa seja responsável por elas. Designe as funções de Analista do Processo de Negócios e de Analista do Sistema para a mesma pessoa - útil quando o contexto de negócios precisa ser entendido, mas a organização não tem as habilidades existentes do Analista do Processo de Negócios. Existem várias habilidades comuns entre essas funções. Um problema é que essa pessoa pode achar difícil separar as necessidades de negócios das soluções possíveis, uma vez que o Analista de Sistemas trabalha com requisitos para sistemas automatizados. Designe as funções de Analista do Processo de Negócios e de Analista de Teste para a mesma pessoa - útil onde os clientes estiverem envolvidos ativamente na definição e na avaliação contínua do projeto. É possível misturar essas funções com alguma equipe tecnicamente mais qualificada designada exclusivamente às funções de Analista do Processo de Negócios e de Analista de Teste.
